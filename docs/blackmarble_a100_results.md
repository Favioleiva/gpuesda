# Black Marble Peru A100 results

## Canonical run

The production run was executed successfully on an **NVIDIA A100-SXM4-80GB** using `gpu_esda 0.2.0`, CuPy 14.0.1, seed 12345, and the VNP46A2 raster for 2024-03-21. The package wheel and statistical algorithms were not changed after the run.

The analyzed variable was:

```text
Y = log(1000 × NTL + 1)
```

The factor 1000 expands fractional radiance before logarithmic compression. The transformation preserves NTL zero as zero and does not convert NoData to zero.

All production inference used 999 permutations. Local significance used Benjamini–Hochberg FDR with `q ≤ 0.05`.

| Rook | 0.6121386781 | 0.001 | 50 | 6,111,908 | 403,205 | 6.60% |

Rook has the largest Global Moran I, indicating the strongest global association under its four-neighbor topology, but it retains substantially fewer local clusters after FDR. Queen and radius-2 `d^-2` have very similar Global Moran I. Nevertheless, `d^-2` identifies 185,155 more significant local clusters than Queen, showing that similar global summaries do not imply the same local inferential classification.

## Raw significance versus inferential clusters

The package result's `significant` array is the raw BH-FDR boolean. It has the raster shape, but it is not by itself the reportable cluster mask. The canonical inferential definition is:

```text
valid & ~island & significant & quadrant_between_1_and_4
```

This intersection guarantees that NoData, structural holes, islands, and invalid quadrant codes never enter HH, LH, LL, or HL counts. For Queen, the raw boolean contains 1,174,898 true values. Exactly 10 are islands, so the definitive inferential count is 1,174,888. The preliminary inclusive count must not be used as the Queen cluster total.

The original execution CSV and JSON remain unchanged provenance artifacts. The corrected, versioned interpretation is provided by:

- `canonical/blackmarble_2024-03-21_log1000p1_a100_comparison_canonical.csv`
- `canonical/blackmarble_2024-03-21_log1000p1_interpretation.json`
- `canonical/artifact_manifest_sha256.csv`

The canonical table is regenerated by `scripts/blackmarble_postprocessing.py`. For Queen it reads the raw `significant` and `quadrant` columns directly from the archived Parquet rather than relying on an in-memory mask that may already have been sanitized.

The four significant-class counts satisfy:

```text
HH + LH + LL + HL = local_FDR_significant
```

For all observed non-island classifications they also satisfy:

```text
HH + LH + LL + HL = n - islands
```

## Moran scatter plots

Each canonical scatter plot uses standardized `Y` horizontally and its spatial lag `WY` vertically. The reference line is the no-intercept identity:

```text
WY = I × Y
```

It therefore passes through the origin and has slope equal to the operator's observed Global Moran I.

## Timing scope

Recorded times describe this one A100-SXM4-80GB execution, its software stack, data, and notebook state. They are not universal hardware benchmarks. The run metadata reports 134.139 seconds for the recorded workflow. The three global 999-permutation stages took 9.395 seconds (Rook), 10.512 seconds (Queen), and 11.594 seconds (`d^-2`, radius 2).

Canonical tables, figures, metadata, and local results are stored under `outputs/blackmarble_peru_2024-03-21_a100/`.

Only a Queen cell-level Parquet was retained. Its two local paths are byte-identical copies of one result. No Rook or radius-2 `d^-2` local Parquet exists in this execution archive. This is an archival limitation, not a missing statistical run: their global results, class counts, and figures are present, and inference was not repeated merely to create additional files.
